# Islands — a slide composition model for Lattice

**Status:** ratified 2026-06-11; **Phase 0 landed** (the `--isl-*` berth
inset token rename). The design model, the maker/checker prototype
(`.scratch/islands/`), and the staged execution plan below. Direction was
confirmed before any engine edit; the remaining phases are queued, not done.

---

## 1. The reframe

Today a Lattice slide is **content + chrome bolted on**:
`section` is a flex column; the eyebrow and title *flow* as the first
content children; header / footer / pagination / logo are absolutely
positioned at **repeated magic-number insets** (`top/bottom: 1.875cqi`,
`left/right: 2.34375cqi`); marks/tints are `::before/::after` pseudos.
There is **no slide-level coordinate system** — every chrome element
re-derives its own berth, and "where does the title sit" is an accident
of flow.

Yet the engine *already* does islands, informally, in one place: the
`split-panels` transform **lifts** the eyebrow + heading + lede out of
flow into named DOM zones (`.panel-left` / `.panel-right`), and every
manifest declares `slots` — CSS-selector-named regions. "Islands" is
just **promoting that proven per-component pattern to the whole slide.**

> **An island is a self-contained region of the slide with its own
> berth (fixed position), z-plane, population source, visibility rule,
> and — critically — a *deterministic px box*. A slide is a sea
> (canvas) plus a fixed set of berths, each holding one island or one
> island *group*.**

The name fits the way Astro uses it: independent regions on a shared
page. It also matches what we already say informally ("the eyebrow
slot", "the chrome", "the stage").

## 2. Three structural ideas

### 2.1 Berths — the named coordinate frame
One token contract replaces the scattered magic numbers. The slide is a
grid of three full-width **bands**, each sub-divided left / centre / right:

```
 z0 canvas · rule  ────────────────────────────────────────────────
┌───────────────────────────────────────────────┬──────────────────┐
│  kicker (eyebrow)              MASTHEAD band    │  logo · status   │  1–2 rows
│  title  (+ lede)               (the slide's     │  meta            │
│                                 masthead)       │                  │
├───────────────────────────────── STAGE ─────────────────────────-─┤
│         MAIN — the content island (a concrete px box)             │  1fr
├───────────────────────────────── FOOTER ──────────────────────────┤
│  footer-left            progress rail            pagination        │  auto
└───────────────────────────────────────────────────────────────────┘
```

The masthead being a **1–2 row band** is the headline structural move:
kicker + title are **lifted out of content flow** into the masthead
berth (same kernel as `liftSlotLabel` / `split-panels`, promoted to the
slide level). MAIN is then body-only and sits in a clean stage.

### 2.2 Islands — the registry
14 islands across four z-planes. Nine ship today (in scattered
mechanisms); five are the **future / missing** set this model gives a
principled home.

| # | Island | z | Berth | Population | Hide token | Today |
|---|--------|---|-------|------------|-----------|-------|
| 1 | canvas | 0 | full | theme `--bg` + dark | — | ✓ |
| 2 | rule | 0 | top edge | `--spectrum` | dark→off | ✓ |
| 3 | atmosphere | 1 | full | `tint-*` / `mark-*` | `treatment-none` | ✓ |
| 4 | watermark | 1 | stage | front-matter / class | — | partial (split-panel only) |
| 5 | kicker | 3 | masthead · stage r1 | `` `eyebrow` `` | `no-kicker` | ✓ but **in flow** |
| 6 | title | 3 | masthead · stage r2 | `## h2` (+ lede) | — | ✓ but **in flow** |
| 7 | **meta** | 3 | masthead · bay | front-matter `meta:` | — | ✗ **new** |
| 8 | logo | 3 | masthead · bay | front-matter `logo:` | `logo-on` | ✓ |
| 9 | **status** | 3 | masthead · bay | state variants | `silent` | partial (tint only) |
| 10 | main | 2 | stage | component DOM | — | ✓ |
| 11 | footer | 3 | footer · left | `footer:` | `no-footer` | ✓ |

The **missing five** are exactly the boardroom gaps we hack around today:
- **meta** — date / owner / classification ("CONFIDENTIAL · Q2'26 Board
  Review"). Today crammed into the footer or the eyebrow.
- **progress** — a section dot-rail / breadcrumb. Orientation in a
  40-slide deck. Today: nothing (pagination is a bare number).
- **status** — draft / confidential / WIP as a *real bay island* that
  composes in the masthead, not a full-bleed tint overlay.
- (**watermark** generalised off split-panel; **annotation** formalised
  as the z4 review plane that is suppressed in delivery.)

### 2.3 Groups — islands compose (this is the **Form** layer made concrete)
A berth holds either a leaf island **or a group**: a composite that
subdivides its berth into sub-islands with their own internal sub-grid
and polarity. **Lattice's existing `Form` vocabulary IS the group
catalog** — `panel`, `split`, `triptych`, `matrix`, `canvas` are group
signatures. Two kinds:

- **Framed group** — sits in the stage berth, shares masthead/footer.
  (e.g. the chart-stage prototype: a caption sub-island + a chart
  sub-island.)
- **Sovereign group** — claims the whole canvas and **suppresses the
  chrome berths**. `split-panel`, `title`, `image-full` are sovereign —
  *that is exactly why a split-panel reads as "distinct from anything
  else".* It isn't a content layout inside the frame; it replaces the
  frame.

Every sub-island is itself `container-type: size`, so a group nests
arbitrarily and each leaf still resolves to a concrete box.

## 3. The concrete-dimension contract (non-negotiable)

Charts and Mermaid need a **parent with concrete dimensions** or they
collapse / overflow. Today's flex-column `main` makes the content box
**content-driven** — its height is whatever the content is, which is the
wrong order of causality for an SVG that must be sized *to its frame*.

The islands grid fixes this by construction: a berth is a **grid track
of a fixed-size slide**, so it computes to px *before* its content lays
out. The contract:

> **Every island resolves to a deterministic px box appropriate to the
> active resolution, independent of its content.** A chart/diagram
> mounts into `stage` (or a stage-group sub-island) and is told its box
> — it never has to grow one. The `chart-stage` prototype shows a bar
> chart filling an exact, known frame; at 4K the same cqi math triples
> the box automatically.

This also means **per-berth fill discipline** (see §5, the checker's
systemic finding): when an island *underfills* its box, the box decides
how it distributes — `center` / `optical-center` / `anchor` — rather
than defaulting to top-left flow.

### The gap + clip contract (islands never touch or bleed)

The dual of "every island is a concrete box" is **every island keeps a
defined gap to its neighbours, and overflow is clipped, not bled.** A
slide that doesn't meet the manifesto's density bar must *clip*, never
spill one island's content across another.

- **Defined gaps.** Each berth carries a guaranteed gap to the next:
  masthead↔body is the band's bottom margin + hairline rule; body↔footer
  is a reserved footer safe-area; the bay stacks its islands on a gap
  token. No two islands share an edge.
- **Clip, don't bleed.** `overflow: hidden` is the rule, not the
  exception. The `<section>` clips at the slide boundary; the masthead and
  bay clip their own overflow (a too-long title or meta line is cut at the
  berth). The **fully-bounded body clip** — the stage as its own hard
  max-height, overflow-hidden box so overrun can't even reach the footer
  berth — lands with the **Phase 4 stage wrapper**; until then the
  footer-reserve gap keeps well-authored content clear, and the section
  clip stops anything escaping the slide.

Shipped in Phase 2b for the masthead/bay/footer-reserve; completed for the
body in Phase 4.

## 4. Prototype (maker → checker → refine)

Six standalone slides at 1920×1080, palette-blind, using the **real
cuoio + base tokens** (zero hand-picked hex), rendered via the
puppeteer chromium. Built by a maker pass, critiqued by an independent
checker against a 10/10 board bar, then refined:

| Slide | Demonstrates | Checker v1 → v2 |
|-------|--------------|------|
| `content-light` | full chrome — all masthead + footer islands | 7 → ~8.5 |
| `cards-light` | MAIN flexibility + **status island** in the bay | 6 → ~8.5 |
| `chart-stage` | **framed group** + the **concrete-box** chart mount | 3 → ~9 |
| `split-panel` | **sovereign group** (feature + support) | 4 → ~8 |
| `divider-dark` | sovereign anchor (centred bookend) | 7.5 → ~8.5 |
| `anatomy` | the labelled berth/island system diagram | — |

Refinements applied from the checker: killed dev-scaffolding on the
chart (real axis + value labels + target line + hero-bar encoding);
moved the review marker from a clipped floating overlay into the
masthead bay as a status island; warmed the espresso dark surfaces;
balanced the KPI deltas and fixed a semantically-backwards red; centred
the split-panel punchline with a ghost section numeral; tightened the
masthead so the lede stops kissing the rule.

## 5. The one thing the checker is still right about

> *The berths are positioned correctly, but content inside them is not
> always **composed** to the berth — it top/corner-anchors by default.*

So the model carries a **berth-fill axis** as a first-class property
(not an afterthought): each berth declares how an underfilling island
distributes. This is what separates "grid of correctly-placed boxes" from
"board deck". It's cheap — one `align/justify` contract per berth — and
it's the difference-maker.

## 6. Execution plan (staged; confirm before engine edits)

**Guiding principle:** ship the *contract and the value* without a
big-bang migration that risks every component. Two mechanisms exist:

- **A — section-as-grid (north star).** `section { display:grid;
  grid-template-areas }`. Cleanest, but every component assumes it is a
  flex child of `section`; wholesale flip breaks them. Large migration.
- **B — berth overlay (incremental).** Keep `section` flex; islands are
  absolutely-positioned berths pinned to one shared `--isl-*` inset
  token set — *exactly how chrome already works*, just named and
  unified. Zero component risk.

**Recommendation: B now → A later.**

**Phase 0 — token contract (no behavior change).** Replace the magic
insets in `marp.scaffold.css` / `base.modifiers.css` with named
`--isl-frame-x/-y`, `--isl-gap`. Pixel-identical; pure rename. Lands the
vocabulary.

**Phase 1 — masthead lift (the high-value move).** A `mastheadLift`
transform (sibling of `liftSlotLabel` — wire into all three render
paths + the registry) pulls the leading `` `eyebrow` `` + first `h2`
(+ optional lede) into a `<header class="isl-masthead">` band. MAIN
becomes body-only. Authoring grammar **unchanged** (still `` `eyebrow`
`` + `## title`). Behind a deck/​section opt-in flag at first; gate on
the per-component galleries (light+dark page counts) + a pixel diff.

**Phase 2 — the missing islands.** `meta:` front-matter → meta island;
a `progress` island driven by `headingDivider` sections; promote
`status` (draft/confidential/WIP) into a bay island. Each is additive,
behind a token, palette-blind.

**Phase 3 — groups as Form.** Formalise `split-panel` et al. as
declared **sovereign/framed groups** in the manifest (`group:` +
`sovereign: true`), so the loader knows a sovereign group suppresses
chrome berths. Reuses the `split-panels` kernel; mostly a metadata +
CSS-tokenisation pass.

**Phase 4 — section-as-grid (north star, optional).** Once every
component renders inside a `MAIN` wrapper (Phase 1 gives us the wrapper),
flip `section` to a real named-area grid and drop the absolute berths.
This is the principled end state; it's safe only after MAIN is a
container, and it pairs with the deferred `@layer` activation work
(see `engineering/cascade.md`).

**Gates at every phase:** the three-renderer parity check, the 53×2
per-component galleries + 12×2 bucket surveys (page-count asserted),
and a `tools/pixel-check.js` before/after on the baseline deck. A
visual change that isn't intended fails a gate, not a reviewer's eye.

## 7. Decisions — ratified 2026-06-11

1. **Migration mechanism — B-now → A-later.** Berth overlay first (zero
   component risk), section-as-grid as the staged north star.
2. **Missing islands — all five greenlit.** `meta`, `progress`,
   `status`, and the generalised `watermark` + formalised `annotation`.
3. **Masthead lift — opt-in per deck/section first**, graduate to
   default after it proves out on a feature deck + the per-component
   galleries (matches the workflow isolation rule).
4. **Doc home** — promote this to
   `engineering/decisions/2026-06-11-islands.md` when Phase 0 starts.

### Execution order, locked
- **Phase 0** — name the magic insets as `--isl-frame-x/-y` / `--isl-gap`
  in `marp.scaffold.css` + `base.modifiers.css`. Pixel-identical rename;
  lands the vocabulary. *(safe first step)*
- **Phase 1** — `mastheadLift` transform (3 render paths + registry),
