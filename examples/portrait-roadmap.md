# A roadmap that reads down the page.

`roadmap · horizons · portrait · 2026-06-19`

A roadmap is a wide grid — workstreams down, phases across. On a tall deck those columns crush. So portrait picks the form the engine already had: the phase cards, stacked into one column you read top to bottom.

---

<!-- _class: content -->

## Same table, the right shape for the box.

You author the same workstream × phase table. The kernel reads the deck's orientation and, on a tall box, transposes it to phase cards and stacks them — no second authoring.

- **Phases become cards**, each carrying its workstream rows.
- **Cards stack** down the page; the header collapses to one line and each row reads `LABEL · value` so three or four phases fit.
- **Landscape is untouched** — the wide table renders exactly as before.
