two ownerships
Chrome tiles — engine-owned (title, meta, pagination, progress…). The author never touches them.
content tile — the engine-owned slot; what the author's markdown flows into.
Component — the author's layout (kpi's hero+supports grid) fills the content tile. Per the model it is itself a Frame → Cells → Tiles (the recursion). This is where portrait reflow lives today.
Read it as: the engine gives the author one Tile — content — and rings it with chrome Tiles (title+kicker lifted to the masthead, meta in the masthead-bay, footer/progress/pagination below). The Component fills the content Tile and, in the model, recurses as its own Frame. That's the seam: orientation/​reflow is a Component concern today (its CSS), which is why "does it work in portrait?" is a per-component fact — and the case for declaring it on the component.
